## Part 1: Data Foundations

### Question 1: What is Data?

Data is basically just raw facts, like a bunch of random numbers or text, that hasn't been organized yet.

**Examples:**

1. A list of temperatures recorded every hour in Nairobi.
2. Number of people who have used e-services of government over a year.

### Question 2: Structured vs. Unstructured Data

**Structured Data:** This is highly organized data that exists in tabular format with interconnected rows and columns, where you can easily search for specific details and single out the relationships between its pieces.

- **Examples:** An Excel spreadsheet of customer contacts that has names,emails,addresses and contacts.

**Unstructured Data:** This is data that does not have a predefined format or organization, which makes it much harder for computers to automatically sort or analyze.

- **Examples:** A collection of PDF documents; a folder full of MP3 voice recordings.

### Question 3: What is a Schema?

A schema is a blueprint or a set of rules that defines how data should be organized

**Example:**
A schema for a book :

- **Title:** A tale of Two Cities
- **Author:** Charles Dickens
- **Price:** Ksh 1,000
- **Year Published:** 1859

**The difference between a List and a Dictionary:**

- **A List** is an ordered collection of items that you can access, update, and iterate over.

- **A Dictionary** is a data structure that stores information in key-value pairs.
